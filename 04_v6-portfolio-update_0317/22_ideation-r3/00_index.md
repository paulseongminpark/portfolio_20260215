# Ideation R3 — 전면 재설계
> 2026-03-18 | FROM: R2 (구조+텍스트 코드 반영 완료)

## 목표
R2 구조의 근본 문제(반복, 사람 부재, 톤 분열) 진단 → 서사 기반 전면 재설계

## 진단 (Opus 비평)

### 4가지 문제
1. **반복**: "맥락이 사라진다"가 6곳에서 변주. mcp-memory/4-Layer도 각 3회.
2. **구조 침범**: About이 System을 선취 → System 도착 시 새 정보 없음.
3. **톤 분열**: 이력서/스펙/고백산문/동기서사 4종 혼재.
4. **사람 부재**: 시스템 소개서. Evolution만 사람이 보임.

### 근본 원인
6개 섹션이 전부 "내가 뭘 만들었나"에 답하고 있었다.
각 섹션이 서로 다른 질문에 답해야 한다.

## 결정

### 포트폴리오의 성격
- 갤러리/이력서+/케이스스터디/에세이/매니페스토 중 → **프로필** (매거진 피처 기사)
- "섹션의 나열"이 아니라 "하나의 서사가 흐르는 긴 스크롤"

### 확정 구조
```
(Hero)           — 이름 + 한 줄 (전체 글 완성 후 결정)
01 · About       — 이 사람이 세상을 어떻게 보는가
02 · Why I Build — 왜 만들게 됐는가 (문제의 경험)
03 · Journey     — 어떻게 여기까지 왔는가 (가장 긴 섹션)
04 · Work        — 뭘 만들었는가 (6개 카드)
05 · Now         — 지금 어떻게 돌아가는가 (시스템 구조 유일 등장)
06 · Forward     — 어디로 가는가
07 · Contact
```

### 순서 결정: Journey before Work
- Journey가 맥락을 깔면 Work 카드가 동기 설명 불필요 → 반복 구조적 차단
- 기각안: Work→Journey (카드가 스스로 여정 설명해야 함 → 반복 유발)

### 반복 방지 매트릭스
| 키워드 | 유일 등장 섹션 |
|---|---|
| 맥락이 사라진다 | Why I Build |
| mcp-memory | Work |
| 4-Layer | Work |
| 세 축 | Now |
| 24→15→3 | Journey |
| 의지가 아니라 구조/조건 | About 또는 Why I Build |
| 루프/측정 | Now |

### AI capability
- About에서 렌즈 세팅, Work에서 증명. 별도 섹션 없음.

### 시각화 (글 완성 후 적용)
- 후보 6개: 밀도 변화, 삭제 시각화, 타이포 스케일, 주석 레이어, 숫자 기념비, 병치
- 1-2개만 선택. 글 먼저, 시각 나중.

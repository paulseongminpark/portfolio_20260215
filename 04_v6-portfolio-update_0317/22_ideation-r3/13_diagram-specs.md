## [다이어그램 6] 승격 경로

**위치**: 설계 4막 4-1, 다이어그램 5 바로 아래
**목적**: 기억이 성숙하는 경로 + 감쇠

**내용**:
- 수평 흐름 (왼→오):
  **Observation** → Gate 1 → **Signal** → Gate 2 → **Pattern** → Gate 3 → **Principle**
- 각 Gate에 조건 표시:
  - Gate 1: SWR threshold 0.25
  - Gate 2: visit_count ≥ 10
  - Gate 3: MDL (Minimum Description Length)
- 아래 방향으로 **감쇠 화살표** (위→아래, 승격의 반대):
  - "effective_strength = base × log(freq) × recency"
  - "안 쓰이면 흐려진다. 죽이지 않되 흐려지게."
- Observation은 빠르게 감쇠 (화살표 굵게)
- Principle은 느리게 감쇠 (화살표 가늘게)

**시각적 톤**: 성장과 감쇠의 공존. 위로 올라가는 힘과 아래로 내려가는 힘.

## [다이어그램 12] Three Layers of AI Interface

**위치**: 강화 6항 "이것이 가리키는 곳" 끝 (포트폴리오 전체의 마지막 다이어그램)
**목적**: 포트폴리오 전체에서 가장 중요한 프레임. "내가 하는 일이 업계 지도에서 어디인가"

**내용**:
- 3층 구조 (아래에서 위로 쌓임, 블록 또는 계단):
  - **Layer 1 (바닥)**: Human → AI = **Prompt Engineering**
    - 설명: "사람이 AI에게 잘 물어보는 법"
    - 연도: 2023~
    - 색: 가장 연한
  - **Layer 2 (중간)**: World → AI = **Context Engineering**
    - 설명: "외부 정보를 AI에게 잘 넣는 법"
    - 연도: Anthropic 2025
    - 색: 중간
  - **Layer 3 (꼭대기)**: Knowledge → AI = **LLM UX**
    - 설명: "지식의 구조 자체를 AI가 추론하기 좋게 설계하는 법"
    - 연도: "아직 이름 없음"
    - 색: 악센트 컬러, 시각적으로 강조
- Layer 3이 다른 두 레이어보다 약간 더 크거나 강조
- 각 레이어 왼쪽에 화살표 방향: Human→AI, World→AI, Knowledge→AI
- 오른쪽에 업계 레퍼런스: 프롬프트 라이브러리/few-shot, RAG/MCP/벡터DB, **mcp-memory/온톨로지 설계**

**시각적 톤**: 선언적, 프레임워크, 포지셔닝. 채용자가 이것만 봐도 이 사람이 뭘 하는지 이해해야 함.
**크기**: 전체 폭, 높이 400-500px. 포트폴리오에서 가장 임팩트 있는 시각 요소.

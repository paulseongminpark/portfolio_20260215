CLS와 mcp-memory의 구조적 대응

상보적 학습 시스템(Complementary Learning Systems, CLS) 이론은 기억이 두 시스템으로 작동한다고 주장한다. 해마(hippocampus)는 개별 경험을 빠르게 저장하고, 신피질(neocortex)은 반복 노출을 통해 천천히 일반화한다.

mcp-memory에서 Session injection이 해마에 대응한다 — 최근 Observation과 Signal을 빠르게 의식에 올린다. 온톨로지의 승격 메커니즘(L0→L2→L3)이 신피질에 대응한다 — 반복적으로 활성화된 패턴만 천천히 상위 구조로 통합된다.

해마의 빠른 학습이 신피질의 기존 구조를 덮어쓰지 않듯, 세션의 최근 데이터가 온톨로지의 Principle을 직접 수정하지 않는다.

작업기억과 세션 컨텍스트

Baddeley의 작업기억 모델 — 중앙집행장치가 음운 루프, 시공간 스케치패드, 일화적 버퍼를 관리한다. 용량은 약 4±1 청크.

LLM의 컨텍스트 윈도우가 작업기억이다. 200K 토큰은 인간보다 넓지만, 무한하지 않다. 그리고 인간 작업기억과 마찬가지로 "무엇을 올리느냐"가 처리 품질을 결정한다. mcp-memory의 injection policy — L2+ 우선, Signal 보조, Observation 현장성 — 는 작업기억에 올릴 청크를 선택하는 중앙집행장치의 구현이다. Context as Currency가 작업기억의 경제학.

양적 차이가 질적 차이를 만드는가? 인간 작업기억은 4개 청크로도 복잡한 추론을 하는데, 이건 청크가 고도로 압축되어 있기 때문이다(전문가의 청킹). 200K 토큰은 양적으로 압도적이지만, 토큰은 압축되지 않은 원시 텍스트다. 따라서 "200K 토큰 = 인간보다 큰 작업기억"이라는 단순 비교는 오해를 낳는다.

더 정확한 대응: injection policy가 200K 토큰을 소수의 고밀도 청크로 조직하는 것이 인간 작업기억의 청킹에 대응한다. L2+ 15개를 주입하는 건 200K를 그대로 쓰는 게 아니라 15개 "청크"로 압축하는 행위다.

스키마 이론과 타입 계층

Piaget의 스키마 이론: 인간은 경험을 기존 스키마에 동화(assimilation)하거나, 스키마 자체를 수정(accommodation)한다.

mcp-memory의 타입 체계(L0-L5)가 스키마에 해당한다. 새로운 경험이 기존 타입에 맞으면 동화(해당 타입으로 저장). 기존 타입으로 분류할 수 없으면 — Unclassified로 보존하거나, 새 타입을 추가한다(Correction이 사후 추가된 것).

schema.yaml의 진화 자체가 accommodation의 기록이다.

BCM 가소성과 승격 게이트

BCM(Bienenstock-Cooper-Munro) 규칙: 시냅스 가중치는 사전 활동 수준에 따라 변하는 수정 임계값(θ_M)에 의해 강화 또는 약화된다. 활성화가 임계값을 넘으면 강화, 밑이면 약화. 이건 frequency-dependent selectivity를 만든다.

mcp-memory의 confidence 필드 + 참조 횟수가 BCM의 단순화된 구현이다. 자주 참조되는 노드의 confidence가 올라가고, 임계값을 넘으면 승격 후보가 된다. 정밀한 수학적 BCM이 아니라, 핵심 원리 — "반복 활성화가 선택적 강화를 낳는다" — 의 실용적 적용이다.

현재 mcp-memory의 confidence 증가는 단조롭다 — 참조될 때마다 올라간다. BCM의 핵심인 "임계값의 동적 변화"(높은 활성화가 임계값 자체를 올려 승격을 더 어렵게 만드는 항상성)는 아직 구현되지 않았다.

따라서 현재 구현은 BCM에서 영감을 받은 단순화된 빈도 기반 모델이 정확한 기술이다. BCM의 핵심 속성은 향후 구현 목표이지, 현재 상태가 아니다.

메타인지와 Correction

메타인지(metacognition)는 "자기가 어떻게 생각하는지에 대해 생각하는 것"이다. Flavell이 1979년에 정의한 이래, 학습의 핵심 요소로 인정받는다.

Correction 타입이 recall 최우선이라는 건 메타인지의 외부화다. "이전에 이런 판단 패턴으로 틀렸다"를 시스템이 자동으로 떠올린다.

인간의 메타인지가 불안정하고 편향되기 쉬운 반면(자기과신, 확증편향), 외부화된 메타인지는 DB에 기록된 그대로 재현된다. 이건 메타인지의 정확성을 높이는 메커니즘이다.

예측 코딩과 역방향 흐름

예측 코딩(predictive coding) 이론: 뇌는 상위 영역에서 하위 영역으로 예측 신호를 보내고, 하위 영역은 예측과 실제의 차이(prediction error)만 상위로 보낸다. 인식은 수동적 수신이 아니라 능동적 예측이다.

mcp-memory의 역방향 흐름 — Principle과 Lens가 Observation의 해석을 규정하는 것 — 이 예측 코딩의 구조적 대응이다. WIM 7렌즈가 세션에 주입되면, 새로운 뉴스를 "이 렌즈의 관점에서 중요한가?"로 먼저 판단한다. 이건 top-down prediction이다. 기존 원칙과 맞지 않는 경험만이 주의를 끌고, 새로운 Pattern으로 승격될 가능성을 갖는다.

다만 예측 코딩의 핵심인 "prediction error만 전파"가 시스템에서 명시적으로 구현되지는 않는다. 모든 새 Observation이 주입되지, "기존 렌즈와 일치하지 않는 것만" 주입되는 건 아니다. 다만 간접적으로: 렌즈와 일치하는 관찰은 "이미 알고 있는 것"으로 처리되어 Decision으로 이어지지 않을 가능성이 높고, 렌즈와 충돌하는 관찰만이 새로운 Decision이나 Failure를 낳는다.

이 간접적 필터링이 prediction error 전파의 약한 근사치다.

기억 응고와 레이어 성숙

기억 응고(memory consolidation): 새로운 기억은 불안정하고, 시간이 지나면서 안정화된다. 해마에서 신피질로의 점진적 전이. 수면 중 재활성화(replay)가 이 과정을 촉진한다.

mcp-memory에서 L0 → L1 → L2의 이동이 응고에 대응한다. 새로운 Observation은 불안정하다 — 아직 패턴인지 노이즈인지 모른다. 시간이 지나고 반복 활성화되면서 Pattern으로 응고된다.

auto-iterate의 주기적 측정이 수면 중 재활성화의 역할 — 축적된 경험을 주기적으로 검토하고 승격 후보를 발견한다.

인지 부하와 Context as Currency

인지 부하 이론(Cognitive Load Theory, Sweller): 작업기억의 용량은 제한적이므로, 불필요한 부하(extraneous load)를 줄이고 본질적 부하(intrinsic load)에 자원을 집중해야 한다.

에이전트 24→15 축소, 스킬 14→9 축소의 경험이 이 이론의 실증이다. 15K 토큰 자동 로딩은 extraneous load — 현재 작업과 무관한 정보가 컨텍스트를 차지한다.

"빼기가 더하기보다 중요하다"는 발견은 인지 부하 이론의 LLM 컨텍스트 버전이다. Context as Currency는 인지 부하 관리의 경제학적 프레이밍.

전문성과 패턴 인식

Chase & Simon의 체스 전문성 연구: 전문가는 개별 기물이 아니라 패턴(chunk)으로 판을 본다. 초보자에게 무작위 배열은 전문가에게도 어렵지만, 실전 배열은 전문가가 압도적으로 잘 기억한다. 전문성의 본질은 "더 많이 아는 것"이 아니라 "더 큰 단위로 보는 것"이다.

L0→L2 승격이 이 과정이다. 개별 Observation은 체스의 개별 기물이다. Pattern은 청킹된 구조다.

전문성은 L2의 밀도로 측정될 수 있다 — Pattern, Framework, Heuristic이 풍부할수록 더 큰 단위로 경험을 처리할 수 있다. mcp-memory의 L2 합계(1,096)는 이 시스템 사용자의 전문성이 얼마나 구조화되었는지를 보여주는 지표다.

이중 과정 이론과 레이어 분리

Kahneman의 이중 과정 이론: 시스템 1(빠르고 자동적)과 시스템 2(느리고 의식적). 일상의 대부분은 시스템 1이 처리하고, 어려운 문제만 시스템 2가 개입한다.

mcp-memory의 세션 주입 정책이 이 이중성을 구현한다. L2+ Principle이 주입되면, 이것은 시스템 2의 역할 — 의식적 판단 규칙. Signal과 Observation의 주입은 시스템 1의 재료 — 빠른 상황 파악. 그리고 Correction의 최우선 주입은 시스템 1의 자동성을 시스템 2가 교정하는 메커니즘이다 — "이 패턴은 자동으로 따르지 마라"는 명시적 경고.

위의 매핑을 더 정밀하게 다듬을 필요가 있다. 직접적 매핑보다는 구조적 유사성으로 접근해야 한다. 시스템1의 "자동성"은 데이터 주입과 직접 대응하지 않는다 — 데이터 주입은 자동 처리가 아니기 때문이다.

더 정확한 대응: session injection policy 자체가 시스템1/2의 분업을 구현한다. L2+ Principle의 자동 주입은 "항상 의식에 올라와 있는 배경 규칙"으로서 시스템1적 역할 — 매번 의식적으로 떠올릴 필요 없이 판단을 구조화한다. Correction의 최우선 주입은 시스템2의 개입 — 시스템1의 자동성을 멈추게 한다. 이 대응은 정밀하지 않으며, 구조적 유비(structural analogy)임을 인정한다.

뇌는 하나, 시스템은 여럿: 분산 인지

인지과학의 분산 인지(distributed cognition, Hutchins): 인지는 개인의 머리 안에서만 일어나지 않는다. 항공기 조종석에서 파일럿, 부조종사, 계기판이 함께 인지를 수행한다. 각 요소가 전체 인지의 일부를 담당한다.

mcp-memory의 12개 프로젝트 에코시스템이 분산 인지의 구현이다. orchestration(조율), mcp-memory(기억), context-cascade(주입), auto-iterate(교정) — 각 시스템이 인지의 다른 측면을 담당한다. 그리고 이 시스템들 사이의 조율이 인지의 질을 결정한다 — Hutchins의 조종석에서 파일럿과 계기판 사이의 소통이 비행의 질을 결정하듯.

이 Lens 전체에서 인지과학 개념과 시스템의 대응은 구조적 유비(structural analogy)이지 동일성(identity)이 아니다. CLS의 해마가 실제로 하는 일과 session injection이 하는 일은 메커니즘이 다르다. BCM의 수학적 모델과 confidence 필드는 정밀도가 다르다. 이 대응들의 가치는 "이 시스템이 뇌와 같다"는 주장이 아니라, "뇌가 해결한 문제와 이 시스템이 해결하려는 문제가 구조적으로 같다"는 통찰에 있다.

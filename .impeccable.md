# Portfolio Design Context

## Users
채용 담당자, 기술 커뮤니티, 동료 엔지니어.
"이 사람이 뭘 하고 있는지, 무슨 사람인지"를 파악하러 온다.
긴 글을 읽을 인내심은 있지만, 처음 3초에 톤을 판단한다.

## Brand Personality
**깊이 · 절제 · 친근** — Anthropic과 Notion 사이 어딘가.

- **깊이**: 표면적이지 않다. 내용의 밀도가 느껴진다.
- **절제**: 필요한 것만 있다. 장식이 없다.
- **친근**: 차갑지 않다. 접근 가능하다.

## Aesthetic Direction
- **톤**: 밝고 깨끗. 라이트 모드 전용.
- **레퍼런스**: draw.io 다이어그램 (C4, UML, flowchart, mindmap, org chart, timeline, fishbone, venn, story map), Anthropic 블로그 미학
- **Anti-reference**: 3D 효과, 그라데이션, 장식적 아이콘, 과도한 색, 차가운 기업 느낌

## Design Principles
1. **색은 의미다** — 장식이 아니라 역할/카테고리/상태를 인코딩한다
2. **플랫 단색** — 그라데이션 없음, 3D 없음, 그림자 없음
3. **2단 텍스트 위계** — 굵은 라벨 + 가벼운 설명. 그 이상 없다
4. **연결선은 조연** — 얇고 중립적(회색). 주인공은 노드
5. **여백이 구조** — 빽빽하지 않다. 공간이 관계를 말한다
6. **형태는 다양, 토큰은 통일** — 다이어그램 종류는 달라도 색·글씨·간격은 같다
7. **대비가 가독성** — 배경과 내용물 사이 명확한 명도 차이

## Diagram Type → Token Mapping
| 다이어그램 유형 | 주요 색 역할 | 형태 |
|---------------|------------|------|
| C4 Container | primary-solid + neutral-solid | 둥근 사각, 경계선 그룹 |
| Flowchart | primary + tertiary + danger | 둥근 사각, 마름모, 타원 |
| Dependency Graph | primary + secondary | 타원/원, 다수 연결 |
| Org Chart / Tree | 계층별 primary→secondary→neutral | 사각 스택 |
| Mindmap | 카테고리별 전체 팔레트 | 방사형, 원+사각 |
| Timeline | accent-solid + neutral | 수평 막대 + 마일스톤 |
| Fishbone | 카테고리별 전체 팔레트 | 뼈대 + 가지 |
| State / Activity | primary + danger + neutral | 둥근 사각, 수영레인 |
| Venn | primary + secondary + tertiary (투명) | 원 겹침 |
| Layer Stack | 레벨별 primary→danger 그라데이션 | 수평 스택 |
| Kanban / Story Map | primary + secondary + danger | 카드 그리드 |

## SVG Helper Components (공유 빌딩 블록)
```tsx
// 모든 다이어그램이 공유하는 기본 요소
Box({ x, y, w, h, variant, label, sub })    // variant: 'primary'|'secondary'|'tertiary'|'accent'|'neutral'|'danger'|'solid-primary'|'solid-accent'
Arrow({ from, to, dashed?, label? })         // 방향 자동 계산
Circle({ cx, cy, r, variant })
Diamond({ cx, cy, size, variant })
Label({ x, y, text, level })                 // level: 'label'|'desc'|'caption'|'hero'
GroupBox({ x, y, w, h, label? })             // surface 색, 점선 경계
```

## page-01 — 에디토리얼 다크 {#page-01}

**컨셉:** 문예지/북 디자인 감각의 다크 포트폴리오

| 항목 | 값 |
|------|----|
| 배경 | `#0f0f0e` (완전한 블랙이 아닌 약간 따뜻한 어두움) |
| 텍스트 | `#f5f0e8` |
| 액센트 | `#c8a96e` (황토/앰버) |
| 타이포 | Noto Serif KR (헤딩) + DM Mono (메타/코드) |

**배치 선택 이유:**

- **세리프 헤딩**: 기술 포트폴리오에서 드문 선택. 오히려 그 낯섦이 기억에 남음. 텍스트가 중심인 포트폴리오임을 즉각 전달.
- **황토 액센트**: 순백 또는 네온이 아닌 따뜻한 금속 색조. 고급 출판물의 감각.
- **DM Mono 메타**: 기술적 배경을 타이포그래피로 암시. 텍스트 레이블, 번호, 날짜에 사용해 리듬감 생성.
- **섹션 구분선**: 컬러 블록 대신 얇은 수평선으로만 구분 → 여백이 호흡을 만듦.

**강점:** 강한 개성, 텍스트 중심 콘텐츠에 최적
**약점:** 이미지/시각 자산 없으면 단조로울 수 있음

## page-02 — OpenAI Minimal {#page-02}

**컨셉:** 극단적 미니멀리즘. 흑백만으로 위계 생성.

| 항목 | 값 |
|------|----|
| 배경 | `#ffffff` |
| 텍스트 | `#0a0a0a` |
| 액센트 | 없음 (흑백만) |
| 타이포 | Instrument Sans (OpenAI 공식 서체) |

**배치 선택 이유:**

- **Instrument Sans**: OpenAI가 채택한 서체로, 이 미학의 레퍼런스를 명확히 함. 가독성과 현대성 모두 충족.
- **색 제거**: 색을 쓰지 않는 것 자체가 디자인 결정. "우리 회사는 컬러가 필요 없을 만큼 자신 있다"는 메시지.
- **여백이 디자인**: padding/margin 자체가 시각 언어. 넉넉한 여백으로 각 요소가 독립적인 존재감.
- **섹션 숫자 레이블**: `01 / 02 / 03` 형태의 번호가 내비게이션 역할 + 디자인 요소 동시 수행.

**강점:** 시간이 지나도 안 늙는 디자인, 어떤 컨텍스트에서도 전문적
**약점:** 차별화 부족 (많이 쓰이는 방식)

## page-03 — Anthropic Warm {#page-03}

**컨셉:** 인간적 따뜻함 + 전문성. Anthropic 브랜드 컬러 인스파이어드.

| 항목 | 값 |
|------|----|
| 배경 | `#f9f6f0` (크림) |
| 텍스트 | `#2c1810` (짙은 갈색) |
| 액센트 | `#c4532a` (Anthropic 오렌지) |
| 타이포 | Libre Baskerville (헤딩) + DM Sans (본문) |

**배치 선택 이유:**

- **크림 배경**: 순백(#fff)보다 눈의 피로도가 낮고, 따뜻한 감각을 주면서도 깔끔함 유지.
- **갈색 텍스트**: 순흑(#000)이 아닌 갈색 → 전체 팔레트와 조화. 인쇄물 질감 연상.
- **Libre Baskerville + DM Sans**: 세리프 헤딩의 권위 + 산세리프 본문의 가독성 대비. 두 서체의 긴장감이 레이아웃에 리듬 생성.
- **오렌지 액센트**: Anthropic의 코퍼 오렌지에서 영감. 따뜻하지만 강한 포인트.

**강점:** AI/테크 업계에서 차별화, 인간적 인상
**약점:** 특정 색 조합 선호도가 갈릴 수 있음

## page-04 — Vercel / Linear Dark {#page-04}

**컨셉:** 개발 인프라 도구 회사 스타일. 테이블로 정보 밀도 극대화.

| 항목 | 값 |
|------|----|
| 배경 | `#000000` (순블랙) |
| 텍스트 | `#ffffff` |
| 액센트 | `#5eead4` (틸/민트) |
| 타이포 | IBM Plex Mono (번호/태그) + IBM Plex Sans (본문) |

**배치 선택 이유:**

- **테이블 레이아웃**: 번호 / 제목 / 기간 / 태그의 4컬럼. Vercel 대시보드, Linear 이슈 리스트에서 차용. 프로젝트를 "관리되는 데이터"처럼 제시.
- **IBM Plex Mono**: 모노스페이스 폰트가 번호와 메타데이터에 쓰이면 정밀감 증가. 기술 친화적 오디언스에게 익숙한 언어.
- **틸 액센트**: 다크 배경에서 틸이 갖는 미래적·기술적 연상. Tailwind의 `teal-300`과 동일 계열.
- **호버 효과**: 테이블 행에 미세한 틸 틴트 → 인터랙티브임을 암시.

**강점:** 기술 역량 강조, 정보 스캔에 최적
**약점:** 개성보다 기능 우선 → 감성적 인상 약함

## page-05 — 에디토리얼 Warm {#page-05}

**컨셉:** 라이트 에디토리얼. 잡지/뉴스레터 텍스처.

| 항목 | 값 |
|------|----|
| 배경 | `#fdf8f3` (따뜻한 오프화이트) |
| 텍스트 | `#2c1810` |
| 액센트 | `#c4532a` |
| 타이포 | DM Sans + 레이블-텍스트 행 패턴 |

**배치 선택 이유:**

- **레이블-텍스트 행 패턴**: `ABOUT`, `WRITING` 등 섹션 레이블이 좌측 고정, 내용이 우측. 신문 레이아웃에서 차용. 스캔과 읽기 모두 지원.
- **섹션 간 구분선**: 컬러 블록 없이 얇은 선만으로 구분 → 텍스트 자체가 무게 중심.
- **따뜻한 팔레트**: page-03과 유사하지만 더 밝고 가볍게. 오전에 읽는 뉴스레터 느낌.
- **Framer Motion fade-up**: inView 애니메이션으로 스크롤 중 읽기 경험 향상.

**강점:** 텍스트 중심 콘텐츠에 적합, 읽기 경험 우수
**약점:** 02(OpenAI Minimal)와 컨셉 유사성 있어 차별화 포인트 설명 필요

## page-06 — Warm Minimal Grid {#page-06}

**컨셉:** 건축/디자인 포트폴리오 감각. 레이블-콘텐츠 2컬럼 그리드.

| 항목 | 값 |
|------|----|
| 배경 | `#f5f0eb` (연한 크림) |
| 텍스트 | `#1a1a1a` |
| 액센트 | `#6b5745` (따뜻한 갈색) |
| 타이포 | DM Sans |

**배치 선택 이유:**

- **좌 200px 고정 레이블**: 섹션 레이블이 내비게이션 역할과 디자인 요소 동시 수행. 건축/인테리어 포트폴리오에서 자주 보이는 패턴.
- **padding: 120~160px**: 과감한 여백이 "여유로움"을 직접 보여줌. 콘텐츠에 집중하게 유도.
- **`DM Sans` 단일 서체**: 웨이트(400/500/700)만으로 위계 생성. 불필요한 서체 혼용 없음.
- **상단 경계선만**: 섹션 하단 경계 없이 자연스럽게 흘러가는 레이아웃.

**강점:** 여백 감각이 세련됨, 디자이너 오디언스에게 호소력
**약점:** 콘텐츠가 적으면 페이지가 너무 비어 보일 수 있음

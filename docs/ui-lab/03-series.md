## 3-a — 잡지 커버 히어로 {#3-a}

**레이아웃:** `#2c1810` 다크 갈색 히어로(전체 화면) → 크림 섹션으로 전환. 세로 텍스트 사이드 장식.

**배치 선택 이유:**

- **첫인상 극대화**: 첫 화면이 어두운 갈색으로 가득 채워지는 것 자체가 차별화. 방문자가 스크롤을 내려야 내용을 볼 수 있음 → 호기심 유발.
- **잡지 표지 감각**: 패션/문화 잡지의 커버처럼 이름과 역할을 대형 서체로. "나는 이 공간의 주인"이라는 자신감 표현.
- **크림으로의 전환**: 히어로에서 본문으로 넘어갈 때 색온도 변화가 시각적 리셋 역할.
- **세로 텍스트**: 좌측 세로 텍스트(PORTFOLIO 2026)가 인쇄물의 책등(spine) 연상.
- **단점**: 히어로가 강렬해서 이후 섹션이 밋밋하게 느껴질 수 있음.

**WorkDetail**: 진한 갈색 히어로 상단 → 크림 섹션 전환 패턴 동일 유지

## 3-b — 타임라인 {#3-b}

**레이아웃:** 수직 타임라인. 오렌지 원형 마커 + 날짜/제목/설명 흐름.

**배치 선택 이유:**

- **경력의 흐름을 시각화**: 프로젝트를 나열이 아닌 "여정"으로 제시. 성장 스토리 강조.
- **오렌지 마커**: 팔레트 액센트를 타임라인 포인트로 활용. 시선의 이동 경로를 명확히 안내.
- **시간 순서 강조**: 개발자가 "어떤 순서로 무엇을 만들었는지"를 직관적으로 파악 가능.
- **LinkedIn/이력서 대안**: 이력서 형식을 시각적으로 아름답게 재해석.
- **단점**: 프로젝트 수가 적으면 타임라인이 짧아 보임. 콘텐츠 밀도 관리 필요.

**WorkDetail**: OVERVIEW → PROBLEM → SOLUTION을 타임라인 노드로 표현

## 3-c — 벤토 그리드 모자이크 {#3-c}

**레이아웃:** 12컬럼 그리드. 카드 크기 가변 (1×1, 2×1, 2×2 등). 현대적 대시보드.

**배치 선택 이유:**

- **Apple, Vercel 등의 Feature 섹션 인스파이어드**: 제품 소개 페이지에서 각 기능을 다른 크기의 카드로 배치하는 패턴. 포트폴리오에 적용.
- **콘텐츠 중요도를 크기로 표현**: 가장 중요한 프로젝트는 큰 카드, 부가 정보는 작은 카드. 위계가 공간으로 표현됨.
- **시각적 다이나미즘**: 일정한 그리드 안에서 크기 변화가 시선을 다양하게 이끔.
- **따뜻한 크림 배경**: 차가운 그리드 레이아웃에 따뜻한 색이 인간적 감각 부여.
- **단점**: 카드 내용과 크기를 맞추는 작업이 수동으로 필요. 반응형 처리 복잡.

**WorkDetail**: 3섹션(OVERVIEW/PROBLEM/SOLUTION)을 크기 다른 벤토 카드로 배치

## 3-d — 좌측 Warm 사이드바 {#3-d}

**레이아웃:** 좌 280px 고정 사이드바(크림 배경 + 오렌지 액센트 네비게이션) + 우측 콘텐츠 영역.

**배치 선택 이유:**

- **문서/앱 레이아웃 패턴**: Notion, 깃허브 위키와 같은 문서 레이아웃. "이것은 잘 정리된 포트폴리오"라는 인상.
- **사이드바에 아이덴티티**: 이름, 역할, 링크, 섹션 목록이 항상 좌측에 노출. 사용자가 헷갈리지 않음.
- **따뜻한 사이드바 배경**: 크림(`#fdf8f3`)을 사이드바에 쓰고 흰 배경과 구분해 공간 분할 효과.
- **활성 섹션 오렌지 표시**: 현재 보고 있는 섹션이 사이드바에서 오렌지로 표시됨.
- **단점**: 좌측 사이드바가 고정되면 콘텐츠 영역이 좁아짐. 정보량이 많은 섹션에서 불리.

**WorkDetail**: 좌측 Warm 사이드바(메타/네비게이션) + 우측 상세 내용 카드

## 3-e — 블로그 아티클 {#3-e}

**레이아웃:** `max-width: 720px` 중앙 정렬. 블로그 포스트 형식.

**배치 선택 이유:**

- **가독성 최우선**: 인간이 편안하게 읽을 수 있는 최적 줄 길이 = 약 65~75자 = 720px 전후. 긴 텍스트 읽기에 최적화.
- **콘텐츠가 주인공**: 레이아웃이 없는 것처럼 투명해지고, 텍스트 자체가 전면에. 글을 잘 쓰는 사람에게 어울리는 포맷.
- **Writing 섹션 강조**: 블로그 형식이 쓰기 능력을 직접 보여주는 컨텍스트. "나는 생각을 글로 잘 표현한다"는 신호.
- **Overreacted, Dan Abramov 블로그 인스파이어드**: 개발자 커뮤니티에서 신뢰받는 레이아웃 패턴.
- **단점**: 포트폴리오로서의 "임팩트"가 약할 수 있음. 처음 보는 사람에게 "단순해 보인다"는 첫인상.

**WorkDetail**: 블로그 아티클처럼 제목 → 메타 → 본문 단락 흐름. max-width 720px 유지.

## 텍스트 설계 원칙

### 1. 설명보다 판단
- 기능 나열 ❌
- 판단 구조 제시 ⭕

### 2. 결과 우선, 배경은 보조
- 두괄식 구조 고정
- "이게 뭔데?" → 즉시 답변
- 과정/배경은 흐름상 필요할 때만

### 3. 단일 원인 배제, 관계/조건 중심 서술
- "사람이 안 써서" → "조건이 맞지 않아서"
- "의지 부족" → "구조적 장벽"
- 환경·맥락·관계를 원인으로

### 4. 과장·마케팅 어휘 제거
- "혁신적", "획기적", "최고" 등 제거
- 보고서 스타일, 절제된 언어
- 수식어 대신 구조 설명

### 5. "왜 이렇게 굴러가지?" 문제의식 유지
- 모든 프로젝트의 출발점
- 관찰에서 시작하는 태도
- 당연한 것을 의심하는 시선

## 프로젝트 공통 흐름

```
1. 관찰
   → 현상/데이터/상호작용 관찰

2. 구조적 문제 정의
   → 사람 문제가 아닌 조건/구조 문제로

3. 조건/규칙 설계
   → 정성 → 정량
   → 데이터 근거
   → 판단 가능한 기준 설정

4. 개입 순서 또는 행동 판단 제시
   → 결과물: 우선순위, 점수, 분류, 추천
```

## Empty House CPS 특이점

### Ontology Layer 구조 확정

**WHY** (목적 계층)
- 빈집 문제의 구조적 특성
- 왜 이 시스템이 필요한가

**WHAT** (정의 계층)
- CPS가 무엇인가
- 가중치 세트 기반 판단 지표
- 개입 우선순위 계산 시스템

**HOW** (구현 계층)
- 데이터 수집
- 지표 설계
- 우선순위 산출

### 다이어그램 용도
- "설명"이 아니라 **판단 구조 시각화** 용도
- 데이터 흐름과 계산 로직을 명시
- WHY → WHAT → HOW 순서로 읽히도록

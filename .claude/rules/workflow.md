# Portfolio 워크플로우 명세

## 원칙
- **What만 받는다**: 사용자가 "무엇을" 말하면 나머지는 이 명세를 따른다
- **최소 범위**: 요청된 것만, 그 이상 변경 금지
- **기록 우선**: 완료 후 METRICS.md에 결과 기록

## 기술 스택 기준
- 현재 스택 확인: package.json 먼저 읽기
- 새 라이브러리 추가 전: 기존으로 해결 가능한지 먼저 검토
- 스타일: 프로젝트 기존 패턴 따르기

## 디자인 기준
- 모바일 퍼스트 (375px → 768px → 1280px)
- 기존 색상/폰트/간격 시스템 유지
- 접근성: alt 텍스트, 키보드 네비게이션, 색상 대비

## 성능 기준
- LCP 2.5s 이하
- 이미지: WebP, lazy loading
- 번들: 불필요한 import 제거

## 작업별 필수 단계

### 새 기능
pf-context → 구현 → pf-reviewer → 배포 시 pf-deployer+security-auditor

### 버그 수정
systematic-debugging → 최소 수정 → code-reviewer

### 디자인 결정
3가지 이상 옵션 탐색 → 비교 → 선택 이유 기록

## 금지 사항
- 요청 범위 밖 리팩토링
- 미검증 라이브러리 추가
- 테스트 없는 배포

FIG. 1 — TWO AXES THAT DEFINE THE FIELD
facet 00

지형 정의: 두 축

AI 메모리/지식 시스템을 두 축으로 분류한다. 축 1: 구조화 정도 — flat memory(텍스트 리스트)에서 layered ontology(계층적 타입 체계)까지. 축 2: 자기개선 여부 — static(수동 유지)에서 self-improving(자동 승격, 자기교정)까지. 이 2×2 매트릭스가 경쟁 지형의 기본 지도다. 그러나 2025-2026에 경쟁 지형이 급변했다 — 여러 경쟁자가 graph 기능을 추가하면서 우하단(structured+static)이 두터워지고 있다.

FIG. 2 — MEM0 FROM FLAT MEMORY TO GRAPH HYBRID
facet 01

Mem0: flat에서 graph-hybrid로 진화 중

초기 모델. Mem0는 대화에서 사실을 추출해 키-값으로 저장했다. “사용자는 채식주의자다”, “선호 언어는 Python이다.” 구조 없음 — 모든 기억이 같은 레벨. 자기개선 없음 — 새 사실이 들어오면 덮어쓴다.

현재 기능 (2026-03). Mem0는 더 이상 단순 flat memory가 아니다. Graph Memory를 추가해 엔티티와 관계를 추출하고 Neo4j/Memgraph/Neptune에 저장한다. 벡터 검색과 그래프 검색을 병렬 실행. LOCOMO 벤치마크에서 OpenAI Memory 대비 26% 정확도 향상, 91% 지연 감소를 주장한다.

로드맵. Conflict Detector + Update Resolver로 모순 지식을 자동 관리. 엔터프라이즈 규모 확장.

Mem0의 graph는 자동 추출 기반 — LLM이 대화에서 엔티티/관계를 추출한다. 이건 편리하지만 추출 품질이 LLM에 의존하고, 관계 타입이 범용적이다. mcp-memory의 관계는 설계된 인지 타입 — crystallized_into, abstracted_from, contradicts 같은 인지적 관계가 8개 군으로 정의되어 있다.

또한 Mem0에는 인지 계층(L0-L5)이 없고, 승격 메커니즘이 없다. Graph는 있지만 “경험이 원칙이 되는 경로”는 없다. 그리고 graph 기능은 Pro 티어($249/월)에 제한된다.

facet 02

Zep/Graphiti: 시간 인식 지식 그래프

초기 모델. Zep은 대화 히스토리를 벡터 DB에 저장하고 검색했다. 장기 기억과 요약을 제공. 그러나 지식의 계층이나 관계 타입은 없었다.

현재 기능. Zep은 “context engineering platform”으로 리포지셔닝했다. Graphiti 프레임워크는 시간 인식(temporally-aware) 지식 그래프를 제공한다 — 모든 엣지에 유효 기간(t_valid, t_invalid)이 있고, 과거 시점의 지식 상태를 재구성할 수 있다. LongMemEval에서 Mem0 대비 15%p 우위(63.8% vs 49.0%).

Graphiti의 강점은 시간 추적이고, mcp-memory의 강점은 인지 계층과 승격이다. Graphiti는 “이 사실이 언제 유효했는가”를 잘 추적하지만, “이 경험에서 어떤 원칙을 추출할 것인가”는 다루지 않는다. 시간적 정확성 vs 인지적 깊이의 차이.

FIG. 4 — COGNEE'S ONTOLOGY-VALIDATED PIPELINE
facet 03

Cognee: 온톨로지 기반 검증

현재 기능. 2026년 2월 $7.5M 시드 투자(Pebblebed 리드, OpenAI/Facebook AI Research 창립자 참여). ECL(Extract-Cognify-Load) 파이프라인으로 지식 그래프 구축. 핵심 차별점: 온톨로지 기반 엔티티 검증 — 도메인 온톨로지를 정의하고, 추출된 엔티티가 온톨로지에 부합하는지 검증. FuzzyMatchingStrategy로 자연어-스키마 불일치를 처리. Memify 파이프라인으로 그래프를 지속적으로 개선.

Cognee가 mcp-memory에 가장 가까운 경쟁자일 수 있다 — 온톨로지를 사용하고, 지식 그래프를 구축하고, 자기 개선을 지향한다. 그러나 Cognee의 온톨로지는 도메인 지식을 다루고, mcp-memory의 온톨로지는 인지 과정을 다룬다. Cognee는 “데이터를 잘 구조화”하고, mcp-memory는 “사고를 잘 구조화”한다.

Cognee에는 인지 계층, 승격, Correction 같은 메타인지적 타입이 없다.

facet 04

추가 경쟁자: Hindsight, LangMem, MemoClaw

Hindsight: 4가지 병렬 검색 전략 + cross-encoder reranking. LongMemEval에서 91.4%(Mem0: 49.0%). 조직 지식(institutional knowledge)에 초점. mcp-memory와의 차이: retrieval 전략의 정교함은 높지만 온톨로지나 승격 메커니즘은 없다.

LangMem: LangChain 생태계. flat 키-값 저장 + 벡터 검색. graph 없음, 온톨로지 없음. 단순성이 장점이자 한계.

LangChain Memory: ConversationBufferMemory, ConversationSummaryMemory 등. 대화의 맥락을 유지하지만, 구조적 지식 축적은 범위 밖이다.

MemoClaw: 극단적 단순성(store/recall 2개 연산), 암호화폐 결제. graph 없음. 개인 프로젝트/간단한 에이전트에 적합.

이들은 모두 “지난 대화에서 뭐라고 했는가”를 기억한다. 한계: “그 기억에서 어떤 구조가 드러나는가”는 답하지 못한다.

facet 05

Letta (구 MemGPT): 가장 자주 비교되는 경쟁자

Letta는 “장기 기억을 가진 에이전트”다. 운영체제의 가상 메모리에서 영감을 받아, 컨텍스트 윈도우를 자동으로 관리한다. 메인 메모리(컨텍스트)와 아카이벌 메모리(외부 저장) 사이에서 데이터를 자동으로 이동한다.

Letta의 초점은 양 — 더 많은 기억을 효율적으로 관리. mcp-memory의 초점은 구조와 질 — 기억을 계층화하고, 패턴을 추출하고, 원칙으로 승격하고, 자기교정한다. Letta는 기억의 양을 늘리고, mcp-memory는 기억의 질을 높인다.

facet 06

PKM 도구와 학술 PKG 연구

PKM 도구. Obsidian, Roam Research, Logseq — 개인 지식 관리(PKM) 도구들. 이들은 구조적이다(링크, 그래프 뷰, 태그). 그러나 AI 통합이 제한적이고, 자기개선 메커니즘이 없다. 사용자가 수동으로 링크하고 분류한다.

mcp-memory는 PKM의 구조적 장점과 AI 메모리의 자동화를 결합한다. 그러나 PKM과 달리 UI가 없다 — MCP 프로토콜을 통한 프로그래매틱 접근만 가능. 이건 현재의 제한이자 설계 선택이다(lean storage 원칙).

학술 Personal Knowledge Graph(PKG) 연구. PKG 분야는 2020년대에 활발해졌다. “개인 지식을 그래프로 구조화하고 AI와 결합한다”는 방향. mcp-memory와 공유하는 문제의식이 있지만, 학술 PKG는 주로 (1) 정보 검색 개선과 (2) 추천 시스템에 초점을 맞추며, 인지 계층이나 자기교정 메커니즘은 거의 다루지 않는다. mcp-memory의 차별점은 PKG를 인지 과정 모델링에 적용한 것이다.

facet 07

OpenAI Memory: 가장 널리 배포된 경쟁자

OpenAI의 Memory 기능: 대화에서 자동으로 기억을 추출하고, 사용자가 수정하거나 삭제할 수 있다. “자기개선”이 있다 — 새 대화에서 기존 기억을 업데이트한다.

추출된 기억은 flat한 텍스트 리스트다. “사용자는 Next.js를 쓴다”와 “사용자는 빼기가 더하기보다 중요하다고 생각한다”가 같은 레벨에 존재한다. 전자는 도구 선호이고 후자는 인지 원칙인데, 이 구분이 없다. 계층 없음, 관계 타입 없음, 승격 경로 없음.

facet 08

우상단 사분면: Structured + Static

Notion AI: 워크스페이스의 문서를 AI가 검색하고 요약한다. 구조가 있다 — 데이터베이스, 관계, 프로퍼티. 그러나 이 구조는 사용자가 수동으로 만들고 유지한다. AI가 구조를 자동으로 발견하거나 승격하지 않는다.

전통적 Knowledge Graph (Neo4j + LLM): 엔티티와 관계를 그래프로 저장한다. 구조적이지만 정적이다 — 새로운 패턴의 자동 발견, 지식의 계층적 성숙, 자기교정은 없다.

학술 연구의 AI Knowledge Graph Reasoning(AKGR)과 Knowledge Graph Question Answering(KGQA)은 대규모 지식 그래프(Wikidata, Freebase)에 LLM을 결합한다. 이들은 공공 지식을 다루고, 범용적이다. mcp-memory는 개인 지식을 다루고, 특수적이다. 학술 KG에는 Belief, Identity, Correction 같은 자기참조적 타입이 없다. 학술 KG가 “세계에 대한 지식”을 다루는 반면, mcp-memory는 “나의 판단에 대한 지식”을 다룬다.

facet 09

빈 공간의 이유 — 그리고 시장 기회와 위험

빈 공간의 이유. structured + self-improving 사분면이 비어있는(또는 극히 희소한) 이유: 세 가지 역량이 동시에 필요하기 때문이다.

1. 엔지니어링: 하이브리드 검색, 그래프 구조, MCP 서버, 세션 주입 파이프라인
2. 인지과학: CLS, 작업기억, BCM 가소성, 메타인지에 대한 이해
3. 철학: 존재론 설계, 자기참조 구조, 들뢰즈적 현실화/반-현실화 개념

이 세 영역의 교차점에 있는 사람이 드물다. 엔지니어는 철학을 모르고, 철학자는 코드를 안 짜고, 인지과학자는 개인 시스템을 안 만든다. Cognee가 온톨로지를 도입하면서 이 공간에 가까워지고 있지만, 인지적 깊이(승격, 역방향 흐름, 자기교정)에서 차이가 있다.

기회. LLM 에이전트가 보편화되면서, “에이전트의 기억을 어떻게 관리할 것인가”가 산업적 문제가 된다. 현재는 Mem0 수준의 flat memory가 주류지만, 에이전트의 작업이 복잡해질수록 구조적 기억의 필요성이 커진다.

위험. (1) OpenAI, Anthropic, Google이 자체 메모리 시스템을 강화하면 독립 솔루션의 공간이 줄어든다. 그러나 이들의 접근은 “플랫폼 내장 메모리”이지 “사용자가 소유하는 인지 구조”가 아니다. 이 차이가 mcp-memory의 지속 가능한 차별점이다. (2) Mem0/Cognee가 인지 계층을 추가하면 차별점 약화. (3) “사용자가 소유하는 인지 구조” vs “클라우드 내장 메모리”의 차이가 일반 사용자에게 소구하지 않을 수 있다.

차별점 재평가 — 그리고 한계

2026-03 경쟁 지형을 반영한 차별점 재평가:

기능 mcp-memory Mem0 Zep/Graphiti Cognee
인지 계층(L0-L5)OXXX
승격 메커니즘OXX▵(Memify)
자기교정(Correction)O▵(Conflict Detector)XX
역방향 흐름OXXX
시간 추적XXOX
온톨로지 검증▵(validators)XXO
로컬 소유O▵(OSS 가능)XO
MCP 표준OXXX
schema-driven extensibilityOXX▵

차별점 요약. 1. 인지 계층(L0-L5): 다른 시스템에 없다. 2. 승격 메커니즘(BCM): 지식이 자동으로 성숙한다. 3. 자기교정(Correction recall 최우선): 실패에서 배운다. 4. 역방향 흐름: 원칙이 경험의 해석을 바꾼다. 5. schema-driven extensibility: 코드 변경 없이 온톨로지 확장. 6. 사용자 소유: 데이터가 로컬 SQLite에 있다. 클라우드 종속 없음. 7. MCP 표준: 특정 LLM에 종속되지 않는다.

이 7가지의 조합이 현재 시장에서 유일하다.

한계. 이 비교는 mcp-memory의 관점에서 선택된 기준이다. 다른 기준(벤치마크 성능, 엔터프라이즈 지원, 커뮤니티 크기, 문서화 품질)에서는 경쟁자들이 mcp-memory를 압도한다. Mem0는 GitHub 40K+ 스타, Cognee는 $7.5M 투자, Zep은 엔터프라이즈 고객 기반을 갖고 있다. mcp-memory는 1인 개발자의 프로젝트다. 이 규모 차이를 무시하면 안 된다. 차별점의 “유일성”은 현재 상태이지 영속적 해자(moat)가 아니다.
